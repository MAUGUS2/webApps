ℹ️ Atenção: Usar apenas target="_blank" pode permitir ataques de tabnabbing, onde a nova aba tenta controlar a aba original.

🛡️ Links com Segurança

Incluímos atributos extras para proteção:

rel="noopener" → evita que a nova aba tenha acesso à aba original via window.opener
rel="noreferrer" → esconde o link de origem (útil para privacidade, mas evite se quiser rastrear o tráfego)

🔎 Google

🧠 Dica bônus: window.opener e ataques de tabnabbing

Quando usamos target="_blank", a nova aba ganha acesso ao objeto window.opener, que representa a aba original. Isso pode permitir que um site malicioso redirecione a aba original para um site falso (ex: tela de login falsa). Esse ataque é conhecido como tabnabbing.

🧨 Exemplo de ataque:
window.opener.location = 'https://site-falso.com';

💡 Solução: Use rel="noopener" para impedir esse tipo de ataque.
